# URL Shortener Service

Высокопроизводительный микросервис для сокращения URL-адресов, написанный на Go. Проект реализует паттерн Clean Architecture и предоставляет JSON HTTP API для создания коротких ссылок и перенаправления (редиректа) на оригинальные ресурсы.

## Технологический стек

* **Язык:** Go (Golang) 1.22+
* **База данных:** PostgreSQL 15
* **Драйвер БД:** `pgx/v5` (выбран за высокую производительность и нативную поддержку пула соединений)
* **Контейнеризация:** Docker, Docker Compose
* **Конфигурация:** `godotenv` (управление через `.env` файлы)
* **Роутинг:** Стандартный `net/http`

## Архитектура решения

Сервис спроектирован с использованием **Слоистой архитектуры (Clean Architecture)**, что обеспечивает слабую связность компонентов и высокую тестируемость кода:

1. **Handler Layer (Слой HTTP):** Обработка входящих запросов, валидация JSON и формирование HTTP-ответов.
2. **Service Layer (Слой бизнес-логики):** Генерация уникальных коротких кодов (алгоритм с базой 62), разрешение коллизий и валидация бизнес-правил.
3. **Repository Layer (Слой данных):** Взаимодействие с PostgreSQL. Логика абстрагирована через интерфейсы, что позволяет легко подменять хранилище.

**Ключевые инженерные решения:**
* **Обработка коллизий:** При генерации существующего кода сервис автоматически делает повторные попытки (Retry Pattern) с ограничением максимального числа итераций (`MaxRetries`).
* **Безопасность параллелизма:** Использование криптографически стойкого генератора или безопасного источника случайных чисел для работы в конкурентной среде.
* **Изоляция конфигурации:** Все настройки вынесены в переменные окружения, что позволяет легко разворачивать проект на разных устройствах.

## Запуск проекта

Для запуска приложения потребуется установленный Docker и плагин Docker Compose V2.

### 1. Подготовка конфигурации
В корне проекта создайте файл `.env` (можно скопировать из `.env.example`) со следующим содержимым:

```env
APP_PORT=8080
BASE_URL=http://localhost:8080/
DATABASE_URL=postgres://postgres:qwerty@db:5432/shortener_db
URL_LEN=6

DB_USER=postgres
DB_PASSWORD=qwerty
DB_NAME=shortener_db
```
### 2. Запуск контейнера
Для запуска нужно, чтобы .env файл лежал в одной директории с docker-compose.yaml. Запустить можно командой
```
docker compose up -d --build
```
### 3. Пример использования
Можно использовать curl для запроса
```
curl -X POST http://localhost:8080/shorten \
    -H "Content-Type: application/json" \
    -d '{"url": "https://www.youtube.com/watch?v=Aq5WXmQQooo"}'
```
### 4. Переход по ссылки
Пример output:
```
{"short_url":"http://localhost:8080/ExR7Nv"}
```
Теперь достаточно вставить полученную ссылку в браузер.
